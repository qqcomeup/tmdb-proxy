# 极简电影控制台设计规格

## 目标

将现有 TMDB Proxy 管理页改造成已确认的“极简电影控制台”：单张电影背景、四项核心状态、底部悬浮操作栏，其余管理信息按需展开。页面保持中文、简洁、炫酷，并继续适配桌面端和手机端。

## 视觉结构

- 使用当前随机背景能力显示单张全屏电影背景，不新增海报墙、影视分类、排行榜或播放功能。
- 通过左深右浅的渐变、底部暗角和轻微噪点确保文字可读。
- 左侧为产品说明和在线状态，避免展示虚构影视内容。
- 右侧半透明状态面板展示四项真实数据：总请求、缓存命中率、实时请求速率、错误率。
- 底部悬浮操作栏保留服务概览、更换背景、查看日志和清理缓存。
- 日志、缓存容量、状态码分布等详细数据放入右侧抽屉，默认收起。

## 功能与数据

- 保留现有 `/admin/status` 登录状态检测和 `/admin/auth` Cookie 会话登录。
- 保留 `/admin/metrics`、`/admin/logs?limit=200` 和 `/healthz` 数据加载。
- 保留 `POST /admin/clear-cache?type=all`，清理时禁用按钮并显示进行状态。
- 保留暂停刷新、自动滚动、日志过滤、复制日志、清空前端日志和退出登录。
- 缓存命中率以 API 与图片缓存命中数合并计算；无请求时显示 `0.0%`。
- 所有日志内容继续使用 `textContent` 创建，不通过 `innerHTML` 注入远端数据。
- 页面仍每 5 秒刷新数据；连接失败时展示非阻塞错误提示并自动重试。

## 响应式设计

- 宽屏使用左右分栏和底部居中的操作栏。
- 小于 900px 时改为单列，状态面板置于说明下方，操作栏保持可横向滚动。
- 小于 640px 时缩小标题和间距，抽屉占满屏宽，日志条目改为纵向排列。
- 背景使用固定遮罩提升移动端文字对比度，不依赖背景图片本身的明暗。

## 可访问性与兼容性

- 所有操作使用原生 `button`，保留键盘焦点样式和可读的 `aria-label`。
- 抽屉支持 Escape 关闭，打开时更新 `aria-expanded`。
- 尊重 `prefers-reduced-motion`，关闭非必要动画和位移效果。
- 不引入第三方前端依赖，不更改服务端接口和 Docker 部署方式。

## 验证标准

- 现有 Node 测试全部通过，且增加页面结构测试覆盖抽屉、背景切换和清缓存 POST 行为。
- `npm run check`、`npm test`、`npm run validate:release` 和 `git diff --check` 全部通过。
- 使用桌面 1440×900 和手机 390×844 截图检查无溢出、文字可读、操作可触达。
- 源码中不出现真实 API Key、管理密钥、Cookie、日志隐私数据或其他敏感信息。
